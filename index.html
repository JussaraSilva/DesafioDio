HTML: o tal do esqueleto da internet

O que é HTML, afinal?

HTML é a sigla de HyperText Markup Language. Traduzindo: é a linguagem que a gente usa pra montar a estrutura de um site — tipo montar o corpo antes de colocar roupa (que seria o CSS, no caso).

Pra que serve?

É com o HTML que a gente diz pro navegador: “ô, isso aqui é um título”, “isso é um parágrafo”, “isso é uma lista” e por aí vai. Ele é fundamental pra organizar tudo certinho na página. Sem HTML, o site é só bagunça visual, uma salada de letras.

Algumas tags que usamos nesse projeto

Nesse desafio, eu usei várias tags que a gente viu nas aulas, olha só:

  • h1 até h6: pra definir títulos, do maior pro menor
  • p: parágrafo, o básico do básico
  • mark: serve pra destacar um pedaço do texto
  • small: texto pequenininho, tipo rodapé
  • i: deixa o texto em itálico
  • u: sublinha assim
  • strong: deixa o texto em negrito com importância semântica
  • ol / ul: listas ordenadas (numeradas) e não ordenadas (com bolinhas)
  • li: cada item da lista
  • a: link pra outra página ou site
  • hr: uma linha pra separar os blocos
  • sub e sup: tipo H2O ou 210
  • blockquote: pra fazer uma citação estilosa
Fui além e testei umas tags novas também:
  1. font: essa muda cor e fonte, mas já tá meio aposentada. Hoje a galera usa CSS pra isso.
  2. del: risca o texto, tipo esse aqui, ótimo pra mostrar o que foi removido ou corrigido.
  3. abbr: aquela tag que mostra o significado ao passar o mouse, tipo CSS.
Resumindo a parada

HTML é o começo de tudo quando a gente fala de criar sites. É simples, mas super importante. Aprender essas tags é tipo aprender a montar a estrutura de uma casa. Depois você decora com CSS e anima com JavaScript, mas se a base tiver torta... já era. Então manda ver no HTML primeiro!

“Não é bonito, não é animado, mas sem HTML nenhum site se sustenta.”
